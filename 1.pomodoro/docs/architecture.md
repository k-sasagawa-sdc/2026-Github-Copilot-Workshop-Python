# アーキテクチャ概要

## 現状（Phase 1 MVP）

現在の実装は MVP フェーズであり、Flask によるHTML配信とブラウザ側JavaScriptによるタイマー制御のみが実装されている。

---

## レイヤー構成

```
1.pomodoro/
├── app.py                  # Flask アプリケーションファクトリ
├── templates/
│   └── index.html          # メイン画面テンプレート
└── static/
    ├── css/
    │   └── style.css       # スタイルシート
    └── js/
        ├── timer.js        # タイマーコアエンジン（PomodoroTimer）
        └── app.js          # アプリケーション初期化・DOM バインディング
```

---

## バックエンド（Flask）

### `app.py`

`create_app()` 関数でFlaskアプリケーションを生成するファクトリパターンを採用している。

- `BASE_DIR` を用いてテンプレートフォルダとスタティックフォルダのパスを解決する
- 現在登録されているルートは `GET /` のみ
- `app = create_app()` によりモジュールレベルでインスタンスを生成する

---

## フロントエンド（JavaScript）

タイマーのカウントダウン処理はすべてブラウザ側で実行される。

### モジュール構成

| ファイル    | 役割                                           |
| ----------- | ---------------------------------------------- |
| `timer.js`  | タイマーエンジン。状態管理と残り時間の計算を担当 |
| `app.js`    | DOMへのバインディングとタイマーの初期化を担当   |

### スクリプト読み込み順

`index.html` で `defer` 属性を使用してスクリプトを読み込む。

```html
<script src=".../js/timer.js" defer></script>
<script src=".../js/app.js" defer></script>
```

`timer.js` が先に読み込まれ `window.PomodoroTimer` を設定し、その後 `app.js` が参照する。

---

## 設計原則

- タイマー残り時間は `setInterval` の単純減算ではなく、**開始時刻と現在時刻の差分**から計算する
- これにより、タブ非アクティブやブラウザ復帰後も正確な残り時間を維持できる
- `timer.js` はブラウザ環境とNode.js環境（テスト）の両方で動作するよう設計されている
