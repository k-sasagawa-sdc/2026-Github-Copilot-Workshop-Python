# 実装する必要のある機能一覧

## 概要

このドキュメントは、ポモドーロタイマー Web アプリケーションを実装するにあたって必要となる機能を整理したものである。

Flask をバックエンド、HTML/CSS/JavaScript をフロントエンドとして利用し、まずは MVP を完成させたうえで、段階的に拡張する前提で機能を洗い出す。

関連する設計方針は architecture.md を参照する。

## 1. タイマーコア機能

### 1.1 タイマー表示

- 現在の残り時間を分と秒で表示する
- 現在のモードを表示する
  - 作業
  - 短休憩
  - 長休憩
- タイマー完了が近いことを視覚的に分かるようにする

### 1.2 タイマー操作

- タイマー開始
- 一時停止
- 再開
- リセット
- 必要に応じてスキップ

### 1.3 タイマー精度の維持

- setInterval の単純減算ではなく経過時刻ベースで残り時間を計算する
- タブの非アクティブ化後に復帰しても時間がずれないようにする
- PC スリープやブラウザ復帰後も正しい残り時間を再計算する

### 1.4 状態管理

- idle 状態を管理する
- focus_running 状態を管理する
- short_break_running 状態を管理する
- long_break_running 状態を管理する
- paused 状態を管理する
- 不正な状態遷移を防ぐ

## 2. ポモドーロ進行機能

### 2.1 作業と休憩の切り替え

- 作業セッション完了後に短休憩へ切り替える
- 所定回数の作業セッション完了後に長休憩へ切り替える
- 休憩完了後に次の作業セッションへ切り替える

### 2.2 周期管理

- 作業セッションの完了回数をカウントする
- 長休憩に入るタイミングを判定する
- 必要に応じてサイクルをリセットする

### 2.3 セッション完了処理

- セッション完了時に状態を更新する
- 次のモードを決定する
- 通知や記録処理を呼び出す

## 3. UI 機能

### 3.1 画面レイアウト

- UI モックに沿ったレイアウトを構築する
- メインタイマー、モード表示、操作ボタンを配置する
- 情報が一目で分かる構成にする

### 3.2 視覚的フィードバック

- 現在モードに応じて見た目を切り替える
- ボタンの有効、無効を視覚的に分かるようにする
- 進捗リングやプログレス表示を反映する

### 3.3 レスポンシブ対応

- デスクトップ表示に対応する
- モバイル表示に対応する
- 画面幅に応じてレイアウトが崩れないようにする

### 3.4 アクセシビリティ

- キーボード操作に対応する
- 視認しやすいコントラストを確保する
- 状態変化を分かりやすく伝える

## 4. 設定機能

### 4.1 タイマー設定

- 作業時間を設定できる
- 短休憩時間を設定できる
- 長休憩時間を設定できる
- 長休憩までの作業回数を設定できる

### 4.2 設定の反映

- 保存した設定を初期表示時に読み込む
- 設定変更後にタイマーへ反映する
- 不正な入力値をバリデーションする

### 4.3 設定保存

- 初期段階ではローカル保存に対応する
- 後続でサーバ保存に対応する

## 5. 通知機能

### 5.1 完了通知

- セッション終了時に通知音を鳴らす
- 必要に応じてブラウザ通知を表示する

### 5.2 モード切り替え通知

- 作業開始時に状態の切り替わりを分かるようにする
- 休憩開始時に状態の切り替わりを分かるようにする

## 6. バックエンド機能

### 6.1 画面配信

- Flask でメイン画面を返す
- テンプレートを通じて初期表示を行う

### 6.2 設定 API

- 設定取得 API を提供する
- 設定更新 API を提供する
- 設定値を検証する

### 6.3 セッション API

- セッション保存 API を提供する
- 完了したセッションの記録を受け付ける

### 6.4 統計 API

- 日次統計を返す
- 週次統計を返す

## 7. データ保存機能

### 7.1 設定データ保存

- 作業時間を保存する
- 休憩時間を保存する
- 長休憩周期を保存する
- 更新時刻を保存する

### 7.2 セッションデータ保存

- セッション種別を保存する
- 予定時間を保存する
- 実績時間を保存する
- 開始時刻を保存する
- 終了時刻を保存する
- 完了可否を保存する

### 7.3 永続化基盤

- SQLite を利用してデータを保存する
- 将来的なデータアクセス層の分離を見据える

## 8. 統計機能

### 8.1 当日統計

- 今日の完了回数を表示する
- 今日の集中時間を表示する

### 8.2 継続状況の可視化

- 週単位の実績を表示する
- 継続傾向を振り返れるようにする

## 9. 開発フェーズ別の実装対象

### Phase 1: MVP

- 単一画面表示
- フォーカスタイマー表示
- 開始、一時停止、再開、リセット
- 基本的な状態管理
- ブラウザ復帰時の時間補正
- 最低限のスタイル適用

### Phase 2: ポモドーロ完全対応

- 短休憩
- 長休憩
- 自動モード切り替え
- 周期管理
- 完了通知

### Phase 3: 永続化

- 設定保存
- セッション記録
- Flask API 実装
- SQLite 連携

### Phase 4: 改善と拡張

- 統計表示
- レスポンシブ最適化
- アクセシビリティ改善
- UI モックへの最終調整

## 10. 優先度の高い機能

このアプリで特に優先度が高いのは次の機能である。

- タイマー精度を保つ機能
- 状態遷移を正しく制御する機能
- 分かりやすい UI 表示

これらが不十分だと、見た目が整っていても実用的なポモドーロタイマーにならないため、最初の実装では最優先で対応する。
